# SpeedCalendar 项目协作注意事项

欢迎加入 SpeedCalendar 项目！为了我们能高效协作，减少代码合并时的冲突，请大家遵守以下约定：

## 1. 项目结构概述

本应用采用现代化的单 Activity 架构，UI 层基于 Jetpack Compose 构建。整个项目结构清晰、模块化，旨在实现高内聚、低耦合。

*   **`com.example.speedcalendar`**: 项目的根包。
    *   **`MainActivity.kt`**: 应用的唯一入口 Activity，负责初始化启动屏和加载主界面。
    *   **`navigation`**:
        *   `Screen.kt`: 定义了应用内所有页面的导航路由，是页面跳转的“地址簿”。
    *   **`ui`**:
        *   `MainScreen.kt`: 应用的主框架，包含了底部导航栏和 `NavHost`，负责管理 `Home`、`AI` 和 `Mine` 三个主模块的切换。
        *   `theme`: 存放所有与 Compose UI 主题相关的代码，如颜色、字体、形状等。
    *   **`features`**: 这是我们最核心的业务功能区域，每个主要功能都以一个独立的包存在。
        *   `home`: **首页**模块，所有与首页相关的界面、逻辑都在这里。
        *   `ai`: **AI**模块，负责处理所有与 AI 相关的功能。
        *   `mine`: **我的**模块，包含用户中心、设置等功能。

## 2. 核心协作原则：按 `features` 功能模块分工

为了最大程度地避免代码冲突，我们将严格按照 `features` 包下的功能模块进行分工。

*   **明确你的“领地”**:
    *   如果你负责**首页**功能，你的所有代码都应该在 `features/home` 包内。
    *   如果你负责**AI**功能，你的所有代码都应该在 `features/ai` 包内。
    *   如果你负责**我的**功能，你的所有代码都应该在 `features/mine` 包内。

*   **如何操作**:
    1.  **创建新页面**: 在你负责的包下创建新的 Composable 函数文件。
    2.  **添加导航路由**:
        *   在 `navigation/Screen.kt` 文件中，为你新创建的页面添加一条新的路由记录。
        *   **【注意】**: 修改此文件前，请先与团队沟通，确保路由命名不会冲突。
    3.  **在模块内部导航**: 在你负责的功能模块内部，你可以自由组织页面的跳转逻辑。

**黄金法则：尽可能只在你负责的 `features` 包内进行修改和添加文件。**

## 3. Git 工作流程：标准功能分支（Feature-Branch）

请**严禁**直接向 `main` 分支推送代码。

1.  **开始新任务前，永远先同步主分支**:
    ```sh
    git checkout main
    git pull origin main
    ```
2.  **为新功能或修复创建你的分支**:
    分支命名应清晰明了，如 `feature/user-profile` 或 `fix/home-screen-bug`。
    ```sh
    git checkout -b <你的分支名>
    ```
3.  **在你的分支上安心开发和提交**。
4.  **推送到远程仓库**。
5.  **创建 Pull Request (PR)**: 在 GitHub (或类似平台) 上，从你的分支向 `main` 分支发起一个 Pull Request (PR)，并指定至少一位同事进行代码审查（Review）。

## 4. 如何处理【共享文件】的修改

有些文件是整个项目共享的，修改它们可能会影响到所有人。**在修改这些文件前，必须在团队群里发起讨论并获得同意。**

这些关键的共享文件包括：

*   **`app/build.gradle.kts`**: 当需要添加或更新整个项目的依赖库时。
*   **`navigation/Screen.kt`**: 当需要为新页面添加全局导航路由时。
*   **`ui/MainScreen.kt`**: 应用的主框架，一般情况下**不应修改**。
*   **`res/values/*`**: 如 `strings.xml`, `colors.xml` 等。
*   **`res/drawable/*`**: 公共的图标和图片资源。
*   **`AndroidManifest.xml`**: 如需添加新的权限或组件。

## 5. 沟通是第一生产力

*   遇到任何不确定的问题，不要猜测，**立即沟通**。
*   在开始一项可能影响他人的较大改动前（尤其是修改共享文件），**务必先在群里同步信息**。

遵循以上约定，我们的协作将会非常高效顺畅。祝大家编码愉快！
